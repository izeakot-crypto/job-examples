## Крок 1: Імпорт workflow в n8n

1. Відкрийте https://n8nletsdo.online
2. Перейдіть в розділ **Workflows**
3. Натисніть **"+"** → **"Import from File"**
4. Виберіть файл:
   ```
   [USER_HOME]\OneDrive\Work.Oki-toki\Monitoring of competitors\workflows\monitoring-competitors-v2.json
   ```
5. Натисніть **"Import"**

## Крок 2: Налаштування Google Sheets

### 2.1 Створення таблиці

1. Створіть новий Google Sheets: https://sheets.google.com
2. Назвіть: **"Competitor Monitoring"**
3. В Sheet1 додайте заголовки (перший рядок):
   ```
   Date | Company | AI Summary | Link
   ```

### 2.2 Отримання Sheet ID

З URL таблиці:
```
https://docs.google.com/spreadsheets/d/1abc123xyz456/edit
                                     ↑
                              Це ваш Sheet ID
```

### 2.3 Налаштування в workflow

У workflow знайдіть і відредагуйте **2 ноди**:

#### Нода "Read Previous Data":
- Відкрийте ноду
- В полі **Document ID** вставте ваш Sheet ID
- В полі **Sheet Name** виберіть "Sheet1"
- Збережіть

#### Нода "Save to Sheets":
- Відкрийте ноду
- В полі **Document ID** вставте ваш Sheet ID
- В полі **Sheet Name** виберіть "Sheet1"
- Збережіть

### 2.4 Налаштування Credentials

1. В n8n: **Credentials** → **Add Credential** → **Google Sheets API**
2. Авторизуйтесь через Google OAuth
3. Виберіть цей credential в обох нодах (Read Previous Data, Save to Sheets)

## Крок 3: Налаштування OpenAI (AI Agent)

### 3.1 Отримання API ключа

1. https://platform.openai.com/api-keys
2. Створіть новий API ключ
3. Скопіюйте (показується тільки один раз!)

### 3.2 Додавання в n8n

1. **Credentials** → **Add Credential** → **OpenAI API**
2. Вставте API ключ
3. Збережіть

### 3.3 Підключення до AI Agent

1. Відкрийте ноду **"OpenAI Chat Model"**
2. В полі **Authentication** виберіть ваш OpenAI credential
3. Перевірте модель: **gpt-4.1-nano** (економна) або **gpt-4o-mini** (краща якість)
4. Збережіть

### 3.4 Перевірка AI Agent промпту

✅ AI Agent вже налаштовано з комплексним промптом для аналізу конкурентів.

**Промпт включає:**
- ✨ Аналіз нових фіч
- ⚠️ Виявлення проблем
- 📰 Суммаризація блогу
- 💬 Інсайти з відгуків
- 🎯 Стратегічні висновки

**Детальна документація:** `AI-AGENT-CONFIGURATION.md`

## Крок 7: Активація розкладу

1. Відкрийте ноду **"Schedule Trigger"**
2. За замовчуванням: щодня о 9:00
3. Якщо хочете змінити:
   - **Cron Expression**: `0 9 * * *` (година, хвилина, день, місяць, день тижня)
   - Приклади:
     - `0 9,18 * * *` - двічі на день (9:00 та 18:00)
     - `0 */6 * * *` - кожні 6 годин
     - `0 9 * * 1` - щопонеділка о 9:00

4. **ВАЖЛИВО:** Активуйте workflow!
   - В правому верхньому куті toggle **"Active"** → ON
   - Workflow тепер буде запускатись автоматично

## Troubleshooting

### Проблема: Нода "Read Previous Data" не знаходить дані
**Рішення:**
- Перевірте, що в Google Sheets є колонка "Company"
- Перевірте, що назви компаній співпадають (case-sensitive!)
- Якщо це перший запуск - це нормально, дані ще немає

### Проблема: "Parse All Data" повертає помилку
**Рішення:**
- Перевірте, що всі 3 ноди (Fetch Website, Blog, Reviews) виконались
- Додайте `|| ''` для захисту від null:
  ```javascript
  const websiteHtml = $('Fetch Website').item.json.body || '';
  ```

### Проблема: AI Analysis повертає помилку "unauthorized"
**Рішення:**
- Перевірте OpenAI API ключ
- Перевірте баланс на https://platform.openai.com/usage
- Спробуйте згенерувати новий API ключ

### Проблема: Telegram не відправляє повідомлення
**Рішення:**
- Перевірте Chat ID (має бути число, наприклад: 123456789)
- Спробуйте відправити `/start` вашому боту
- Перевірте токен бота через @BotFather

### Проблема: Workflow занадто довго виконується
**Рішення:**
- Це нормально для 19 компаній (може триваті 10-15 хвилин)
- Можна зменшити timeout в HTTP Request нодах (зараз 30 секунд)
- Можна обробляти по 5 компаній за раз

### Проблема: Loop Companies зациклений (ВИПРАВЛЕНО)
**Симптоми:**
- Workflow не завершується
- Telegram повідомлення не приходить
- Workflow обробляє компанії по колу нескінченно

**Статус:** ✅ ВИПРАВЛЕНО 19.11.2025
- Loop Companies тепер має 3 правильно налаштовані виходи
- Output 2 веде до Prepare Telegram Message після обробки всіх компаній
- Якщо проблема повторюється - перевірте з'єднання в редакторі workflow
