## 📝 Системний промпт

### Main Prompt (1363 символів)

```
Ти - аналітик конкурентів для contact center / VoIP компанії Oki-toki.

ТВОЄ ЗАВДАННЯ: Проаналізуй надані дані про конкурента та створи детальний звіт.

ВХІДНІ ДАНІ:
- company: назва компанії
- url: веб-сайт
- currentData: поточні дані (website, blog, reviews)
- previousData: дані з останньої перевірки (може бути null при першому запуску)

ФОРМАТ АНАЛІЗУ:
✨ НОВІ ФІЧІ:
- Порівняй поточні дані з попередніми
- Знайди нові функції, продукти, інтеграції
- Якщо previousData = null, опиши основні фічі сайту
- Якщо немає змін: "Нових фіч не виявлено"

⚠️ ПРОБЛЕМИ:
- Аналізуй відгуки (reviews) на наявність скарг
- Технічні проблеми (помилки, баги)
- Скарги на ціни, підтримку, функціонал
- Якщо немає: "Проблем не виявлено"

📰 НОВИНИ/БЛОГ:
- Суммаризуй нові статті з blog.recentArticles
- Порівняй дати з previousData
- Виділи ключові теми статей
- Якщо немає нових: "Нових публікацій немає"

💬 ІНСАЙТИ З КОМЕНТАРІВ:
- Аналізуй reviews.samples
- Що подобається клієнтам
- Які болі/потреби клієнтів
- Якщо немає відгуків: "Відгуки не знайдено"

🎯 СТРАТЕГІЧНІ ВИСНОВКИ:
- Що ми можемо взяти на замітку
- Які їхні сильні/слабкі сторони
- Рекомендації для Oki-toki

ВАЖЛИВО:
- Будь конкретним: вказуй назви фіч, дати статей, цитати відгуків
- Пиши українською мовою
- Використовуй емодзі для структури
- Обсяг: 300-500 слів
- Якщо даних мало - напиши що вдалось знайти
```

### System Message

```
Ти - професійний аналітик ринку VoIP та contact center рішень.
Твоя задача - аналізувати конкурентів компанії Oki-toki та надавати
стислі, але інформативні звіти.
```

## 🔧 Технічні налаштування

### AI Model
- **Model**: gpt-4.1-nano (економна модель)
- **Provider**: OpenAI
- **Credentials**: Oki-toki API GPT
- **Type**: LangChain Agent (версія 2.2)

### Input Data Structure

AI Agent отримує дані з ноди "Parse All Data":

```javascript
{
  company: "Ringover",
  url: "https://www.ringover.com/",
  currentData: {
    website: {
      title: "VoIP Solution for Business",
      description: "Cloud phone system...",
      h1Tags: ["Main Heading", ...],
      hasNews: true,
      hasBlog: true,
      hasPricing: true,
      hasFeatures: true
    },
    blog: {
      articlesFound: 5,
      recentArticles: [
        {
          title: "New AI Features",
          date: "2025-11-15",
          preview: "We are excited to announce..."
        }
      ]
    },
    reviews: {
      found: true,
      count: 3,
      samples: ["Great product!", "Support is amazing", ...]
    },
    scrapedAt: "2025-11-19T12:00:00.000Z"
  },
  previousData: {
    date: "2025-11-18",
    summary: "Previous AI analysis..."
  }
}
```

### Output Format

AI Agent повертає текстовий аналіз який потім форматується нодою "Format for Sheets":

```
✨ НОВІ ФІЧІ:
- Запустили інтеграцію з Salesforce
- Додали mobile app для iOS та Android

⚠️ ПРОБЛЕМИ:
- Клієнти скаржаться на високі ціни
- Деякі відгуки про баги в mobile app

📰 НОВИНИ/БЛОГ:
- "New AI Features" (15.11.2025) - описують AI call analytics
- "Pricing Update for 2025" (12.11.2025) - зміни в цінах

💬 ІНСАЙТИ З КОМЕНТАРІВ:
- Клієнти хвалять якість підтримки
- Просять додати більше інтеграцій

🎯 СТРАТЕГІЧНІ ВИСНОВКИ:
- Активно розвивають AI функціонал - нам варто звернути увагу
- Висока ціна може бути їхньою слабкістю
- Рекомендація: підкреслювати наше співвідношення ціна/якість
```

## 🎯 Оптимізація промпту

### Якщо результати не задовольняють:

#### 1. **Занадто загальні відповіді**

Додай в промпт:
```
КОНКРЕТНІ ПРИКЛАДИ:
- Назви конкретні фічі (не "нові функції", а "AI Call Analytics")
- Цитуй реальні відгуки (не "клієнти незадоволені", а "Reviews: 'Support is slow'")
- Вказуй дати публікацій статей
```

#### 2. **Занадто довгі звіти**

Змін в промпті:
```
- Обсяг: 300-500 слів → Обсяг: 200-300 слів (коротко!)
```

#### 3. **Не порівнює з попередніми даними**

Додай:
```
КРИТИЧНО ВАЖЛИВО:
1. Якщо previousData існує - ОБОВ'ЯЗКОВО порівняй дати, контент, фічі
2. Виділяй тільки НОВІ речі (яких не було в previousData)
3. Якщо нічого не змінилось - чітко напиши "Нових фіч не виявлено"
```

#### 4. **Англійська мова в відповідях**

Підсили в промпті:
```
МОВА: Виключно українська! Навіть назви фіч перекладай або пиши в лапках.
Приклад: "Додали функцію 'AI Call Analytics' (аналіз дзвінків за допомогою AI)"
```

## 🔄 Зміна моделі AI

### Поточна модель: gpt-4.1-nano

**Плюси:**
- Дешева (~$0.01 на компанію)
- Швидка
- Підходить для простих завдань

**Мінуси:**
- Може бути менш точною
- Короткий контекст

### Альтернативні моделі:

#### gpt-4o-mini (рекомендовано)
- Ціна: ~$0.03 на компанію
- Краща якість аналізу
- Довший контекст
- **Як змінити**: В ноді "OpenAI Chat Model" → Model → вибери `gpt-4o-mini`

#### gpt-4o (найкраща якість)
- Ціна: ~$0.15 на компанію
- Найкраща якість
- Розуміє складні контексти
- **Коли використовувати**: Якщо потрібен глибокий аналіз

#### claude-3-5-sonnet (Anthropic)
- Ціна: ~$0.10 на компанію
- Відмінна якість аналізу
- Довгий контекст (200K tokens)
- **Як підключити**: Додай Anthropic credentials → Змінити ноду на "Anthropic Chat Model"

## 📈 Моніторинг якості

### Як перевірити якість аналізу:

1. **Тестовий запуск на 1-2 компаніях**
   - Execute Workflow для однієї компанії
   - Перевір вихід AI Agent
   - Оціни чи відповідає формату

2. **Порівняння з реальністю**
   - Вручну зайди на сайт конкурента
   - Порівняй чи AI правильно визначив фічі
   - Перевір чи відповідають дати в блозі

3. **Перевірка previousData порівняння**
   - Запусти workflow двічі підряд
   - В другому запуску має бути "Нових фіч не виявлено"
   - Або має вказати конкретні зміни

## 🛠️ Troubleshooting

### Проблема: AI повертає "Error" або пусту відповідь

**Причина**: Неправильний формат даних або помилка в промпті

**Рішення:**
1. Перевір вихід "Parse All Data" - чи є всі поля
2. Подивись execution log в AI Agent ноді
3. Спробуй простіший промпт для тесту

### Проблема: AI не використовує previousData

**Причина**: Дані не передаються або промпт не акцентує на цьому

**Рішення:**
1. Перевір що "Read Previous Data" працює
2. В промпті додай: "КРИТИЧНО: Порівнюй currentData з previousData!"
3. Перевір що previousData не null

### Проблема: Занадто дорого (витрати на OpenAI)

**Рішення:**
1. Змінити модель на дешевшу (gpt-4.1-nano → gpt-3.5-turbo)
2. Скоротити промпт (менше інструкцій = менше токенів)
3. Зменшити частоту запусків (1 раз на день → 1 раз на 2 дні)
4. Фільтрувати компанії (аналізувати тільки high priority)
