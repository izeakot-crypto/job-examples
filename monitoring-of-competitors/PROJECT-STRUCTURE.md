### 📁 config/ - Конфігураційні файли

#### companies.json
**Призначення:** Список всіх компаній для моніторингу.

**Формат:**
```json
{
  "companies": [
    {
      "name": "Назва компанії",
      "url": "https://example.com/",
      "region": "Ukraine",
      "priority": "high",
      "monitoring": {
        "website": true,
        "blog": true,
        "socialMedia": {...}
      }
    }
  ]
}
```

**Як редагувати:**
1. Відкрити файл в текстовому редакторі
2. Додати/видалити/редагувати компанії
3. Зберегти файл
4. Workflow автоматично використає оновлений список

**Priority levels:**
- `high` - найважливіші конкуренти (моніторити частіше)
- `medium` - важливі конкуренти
- `low` - другорядні конкуренти

---

#### content-parsers.json
**Призначення:** Налаштування парсерів для різних типів контенту.

**Що містить:**
- CSS селектори для блогів
- Патерни для парсингу новин
- API endpoints для соцмереж

**Коли редагувати:**

- При додаванні нових типів контенту
- При зміні структури сайтів конкурентів

### 🔄 workflows/ - n8n Workflows

#### monitoring-workflow.json
**Призначення:** Основний workflow для автоматичного моніторингу.

**Що робить:**
1. Запускається за розкладом (Schedule Trigger)
2. Читає список компаній
3. Для кожної компанії:
   - Збирає дані з сайту
   - Парсить блог
   - (опційно) Збирає дані з соцмереж
   - Аналізує через AI
   - Порівнює з попередніми даними
   - Зберігає в Google Sheets
   - Відправляє нотифікації про зміни

**Як імпортувати:**
1. n8n → Workflows → Import from File
2. Вибрати цей файл
3. Налаштувати credentials

**Компоненти workflow:**
- Schedule Trigger (запуск за розкладом)
- Read/Parse Companies (читання списку)
- HTTP Request nodes (збір даних)
- Code nodes (парсинг контенту)
- OpenAI node (AI аналіз)
- Google Sheets node (збереження)
- Telegram node (нотифікації)
- IF nodes (логіка та фільтри)
